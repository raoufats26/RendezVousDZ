🎯 PRODUCT OVERVIEW
=================================================
RENDEZVOUSDZ is a smart queue management system for small Algerian businesses.

Actors:
1. Business Owner (authenticated)
2. Public Client (no login)

Core Features Completed:
- Authentication system (register, login, logout)
- Email verification on register
- Forgot password / Reset password (token-based, hashed)
- Business creation (1 business per owner)
- Daily queue logic (1 queue per day per business)
- Walk-in client support (owner adds manually from dashboard)
- Public booking page (/b/<business_id>) — clients join themselves
- Daily client limit enforcement
- Status management: waiting / done / skipped
- Validation and duplicate protection
- Real-time queue updates using Flask-SocketIO
- Business settings page (name, category, city, max clients)
- Analytics page (/analytics) with Chart.js charts
- Digital Display Mode (/display/<business_id>) — fullscreen TV/tablet view
- Average service time calculation
- Estimated wait time on public booking page
- Dark mode (persisted via localStorage key: rvdz_theme)
- Language switcher: EN / FR / AR (persisted via localStorage key: rvdz_lang)
  - Arabic triggers RTL layout
  - All UI strings translated via data-i18n attributes + TRANSLATIONS object in main.js
  - switchLanguage(lang) and toggleTheme() are global functions

=================================================
⚙️ BACKEND ARCHITECTURE DETAILS
=================================================

app.py:
- Initializes Flask
- Initializes Flask-SocketIO (async_mode='threading')
- Registers blueprints: auth_bp, booking_bp, analytics_bp, display_bp
- Socket events: connect, disconnect, join (room = business_<id>)
- Flask-Mail configured via env vars (optional, graceful fallback)
- Rate limiting via Flask-Limiter
- i18n shim: _() function passed to Jinja globals

routes/auth.py:
- /register (POST) — creates user, sends verification email
- /login (POST) — checks verified flag
- /logout
- /verify/<token> — email verification
- /forgot-password (POST) — sends reset email
- /reset-password/<token> (GET/POST) — token hashed with SHA-256, expires 1h

routes/booking.py:
- /dashboard — owner's main view
- /create-business — one-time setup
- /settings (GET/POST) — update business info
- /add-client (POST) — walk-in, emits queue_updated
- /mark-done/<id> — emits queue_updated
- /mark-skipped/<id> — emits queue_updated
- /b/<business_id> (GET/POST) — public booking, emits queue_updated

Real-time logic:
    emit_queue_update(business_id, queue_id)
    Emits 'queue_updated' to room: business_<business_id>
    Payload includes: queue_entries, current_count, max_clients, queue_full

routes/analytics.py:
- /analytics — renders analytics.html with chart data
- /analytics/api — JSON endpoint for future live refresh
- get_analytics_data(business_id) — queries: daily counts (7d), peak hours (30d),
  weekly performance (4w), summary stats, busiest day of week

routes/display.py:
- /display/<business_id> — public fullscreen display (no login)
  Renders display.html with entries + lang from business settings
  Updates via SocketIO (same room as dashboard)

=================================================
🔄 REAL-TIME ARCHITECTURE
=================================================

1. Client/owner browser connects via Socket.IO
2. Emits 'join' with { business_id }
3. Server joins room: business_<id>
4. On any queue change (add/done/skip/public booking):
   emit_queue_update() fires
5. All connected clients (dashboard + display page) update DOM live

Real-time MUST NOT be broken. Do not touch emit logic unless explicitly asked.

=================================================
🎨 FRONTEND ARCHITECTURE
=================================================

All JS lives in static/main.js:
- Theme system: _applyTheme(theme), toggleTheme() → window.toggleTheme
  Saves to localStorage key 'rvdz_theme'
  Applied immediately on script load (before DOMContentLoaded) to prevent FOUC
  Swaps logo: auth-logo / dashboard-logo → logo_white.png (dark) or logo_blue.png (light)

- Language system: _applyLanguage(lang), switchLanguage(lang) → window.switchLanguage
  Saves to localStorage key 'rvdz_lang'
  Supported: 'en', 'fr', 'ar'
  Arabic sets dir="rtl" on <html>
  Translates all [data-i18n="key"] elements on the page
  Translates placeholders via [data-i18n-ph="key"]
  Marks active lang button via class 'lang-active' on .lang-picker buttons

- TRANSLATIONS object in main.js: contains all string keys for en/fr/ar

- Real-time: Socket.IO loaded dynamically, joins room, listens for 'queue_updated'

All templates include ONE .global-controls div (top-right fixed):
    <div class="global-controls">
        <div class="lang-picker">
            <button onclick="switchLanguage('en')" data-lang="en">EN</button>
            <button onclick="switchLanguage('fr')" data-lang="fr">FR</button>
            <button onclick="switchLanguage('ar')" data-lang="ar">ع</button>
        </div>
        <button class="theme-toggle" onclick="toggleTheme()">
            <span class="theme-icon theme-icon-light">🌙</span>
            <span class="theme-icon theme-icon-dark">☀️</span>
        </button>
    </div>

CSS for .global-controls lives at the bottom of static/style.css.
display.html does NOT have global-controls (fullscreen TV mode).
